# Portfolio Project

## About
Portfolio website for a Designer targeting headhunters.

The website should function as a curated visual archive. It must be minimal, editorial, and image-focused. The work should always be the primary focus.

## Stack
- Plain HTML and CSS only
- No frameworks
- No libraries
- No build tools
- No JavaScript unless explicitly requested

## Structure
- One main scrollable landing page
- Separate HTML pages for each project

Main page sections:
1. Landing
2. Projects
3. Contact

## Layout Rules

### Landing
- Light, museum-like background
- Multiple abstract images arranged in an intentional, slightly random editorial composition
- Airy, cinematic, archival feeling
- Small text element placed in top-right corner (not dominant)
- Placeholder title text

### Navigation
- Top-left three-line icon
- Click expands a minimal navigation panel
- Must include:
    - Category 1
    - Category 2
    - Category 3
    - Category 4
    - Contact
- Include placeholder project names under each category

Example:
- Category 1
    - Project 1
    - Project 1.1
    - Project 1.2
- Category 2
    - Project 2
    - Project 2.1
    - Project 2.2

### Projects
Categories and counts:
- Category 1: 3 projects
- Category 2: 1 project
- Category 3: 4 projects
- Category 4: 10 projects

Each project preview:
- Title
- One-line placeholder description

### Layout by Category
- Category 1 → grid
- Category 4 → grid
- Category 2 → embedded video layout
- Category 3 → embedded video layout

Constraints:
- Same-size preview blocks
- No visible divider lines
- Separation through spacing only

### Project Detail Pages
Each project page must include:
- Title
- Year
- Category
- Overview
- Tools
- Process
- Final Outcome
- Images/Video

Use placeholder text where needed.

## Typography
- Serif typeface only
- Sharp, contemporary, restrained
- Not flashy or decorative
- Clear hierarchy
- Smaller titles

## Visual Direction
The site must feel:
- Minimal
- Editorial
- Museum-like
- Cinematic
- Archival
- Experimental
- Airy
- Curated

## Color
- Mostly light background
- Neutral palette
- Subtle accent usage only
- Avoid heavy color blocks

## Contact
Bottom section only:
- Placeholder contact items

Keep minimal.

## Rules
- Only build what is explicitly requested
- Ask before restructuring files or folders
- Do not add new sections or features
- Do not refactor existing code unless asked
- Do not invent content beyond placeholders
- Keep everything editable
- Every decision must serve a professional portfolio audience

## Before Coding
Always ask:
1. Placeholder project naming confirmation
2. Which files to edit
3. Current file structure
4. Menu behavior (overlay or push)
5. Image locations
